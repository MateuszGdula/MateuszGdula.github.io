ez.Slider documentation

ez.Slider is a small JavaScript library intended for dynamic creation sliders in an easy and pleasant way. It’s based on DOM and CSS3 styles in JavaScript syntax notation

1. ez.Slider structure

ez.Slider is simply a JS script that dynamically transforms a valid DOM structure into a slider. It creates interface components based on configuration passed to it. The structure of the ez.Slider is presented in the picture below:

ez.Slider interface components:

  • slidesWrapper - this is an internal <div> container for slides
  • previous/nextSlide - this is a <div> represent previous/next slide button
  • previous/nextSlideRange - this is a <div> component which trigger a previous/next slide on click
  • markersContainer - this is a <div> container of dots slide markers
  • markers - this is a <div> elements which shows the currently displayed slide

HTML5 DOM structure of the ez.Slider is presented below:

Example 1: HTML5 DOM structure of the ez.Slider

 <div class="ezSlider">
  <div class="slidesWrapper">
   ..slides..
  </div>
  <div class="previousSlideRange">
   <div class="previousSlide"></div>
  </div>
  <div class="nextSlideRange">
   <div class="nextSlide"></div>
  </div>
  <div class="markersContainer">
   ..markers..
  </div>
 </div>

2. Use of the ez.Slider

2.1. Adding ez.Slider to code

To add an ez.Slider library to your code, place the link to the library in the <script> tag as an src attribute. You can also download the code and place it on your own hosting.

Example 2:

 <script src="https://mateuszgdula.github.io/ezslider1.0.0/dist/ezslider.min.js"></script>
 <script src="./example/dist/ezslider.min.js"></script>

The ez.Slider can be used in two following ways depending on the need to change the slider interface.

2.2. Use by marking valid DOM elements structure as “ezSliderTarget” class.

Valid ez.Slider target structure must meet the following requirements:

  • it must be built from a container (<div> tag in the Example 1) and 2 or more slides (<img> tags in the Example 1);
  • acceptable tags for container are <div>, <section>, <aside>, <article>, <header> and <footer>;
  • acceptable tags for slides are <div>, <img>, <section>, <aside>, <article>, <header> and <footer>;

Example 3: exemplary valid object for ezSliderTarget class

 <div class=”ezSliderTarget”>
  <img ...>
  <img ...>
  <img ...>
 </div>

The interface of the slider created as ez.Slider target class method is based on CSS3 styles contained in DEFAULT_CONFIGS object and can’t be changed. Use ez.Slider by marking HTML element as ezSliderTarget rebuild internal structure of this element so it’s not needed to append it to the document.

2.3. Use by ez.Slider global object.

Global object ezSlider accept two following arguments:

  • DOMObj - is valid DOM elements structure;
  • configs - is valid configuration object;

Example 4: use of the ez.Slider by ivoking ezSlider global object

 let DOMObj = document.querySelectorAll(“.slide”);
 let configs = {..};

 let slider = ezSlider(DOMObj, configs);
 document.body.appendChild(slider);

Valid DOMObj structure must meet the following requirements:

  • it must be built from a container (<div>, <section>, <aside>, <article>, <header> or <footer> are container acceptable tags) and 2 or more slides (<div>, <img>, <section>, <aside>, <article>, <header> and <footer> are acceptable slides tags);
  • or it must be a NodeList of 2 or more slides with acceptable tags;

Valid configuration object must meet the following requirements:

  • it must contain pairs of keys and values just like a standard JS object;
  • the keys must comply with the requirements ez.Slider validation methods;
  • only valid keys are allowed in ez.Slider configuration object;
  • valid keys and values pairs are listed in the ez.Slider configuration object section;

Use ez.Slider by invoking global object build slider from elements passed as an argument and return the DOM structure, so there is a need to append it to the document (e.g by appendChild method).

3. ez.Slider configuration object

The ez.Slider configuration object is intended for customize slider interface CSS3 styles and slides animation options. For costumize ez.Slider options pass a configuration object to ez.Slider global object as a second argument. Acceptable keys and values of configuration object in current ez.Slider version is listed below.

Valid keys and values of the ez.Slider configuration object
Valid keys Valid values

sliderStyle String slider style name, latest ez.Slider version accepts “shift”, “fade”, and “cover” as a sliderStyle; sliderStyle value sets the animation of the slides inside slider;

autoSlide Boolean autoSllide triggers the next slide at specified intervals;
autoSlideIntervals Number number of autoSlide intervals in seconds;

sliderContainer Object object contains pairs of keys and values of CSS3 styles in JavaScript syntax notation; the styles will be assigned to slider interface components; this object is not subject of validation process, so invalid CSS3 styles will generate bugs of the slider interface; keys ends with "Hover" contain styles displayed when mouse is over this element; markerActive contains styles for marker representing displayed slide;

 let configs = {
  sliderStyle: “cover”,
  slideMarkers: false,
  nextSlideStyles: {display: “none”},
  previousSlideStyles: {display: “none”}
 };

4. Default configs of the ez.Slider.

All of the data passed to ezSlider in configuration object are completed by default configs. If the configuration object is valid then it is mixed up with default configs. The only way to change a particular default style is to overwrite it in configuration object passed to ez,Slider (you can also change it in DEFAULT_CONFIGS.js file but it isn’t recommended). Every component of ez.Slider interface has its own default styles. The only exception is the width and height of the slider, these parameters always adapt to the slides size.
